## Общая информация о проекте
ML-пайплайн предсказания высоты зданий создан в рамках решения кейс-чемпионата Chaengellenge >> Cup IT 2026
Проект создан для объединения геоданных из нескольких источников, очистки и сопоставления геометрии зданий, а также предсказания высоты зданий с помощью ML-моделей без data leakage для города Санкт-Петербург.

Основной сценарий:
- загрузка данных из нескольких источников;
- очистка геометрии и атрибутов;
- пространственное сопоставление объектов;
- выбор или агрегация целевой высоты;
- обучение модели для зданий без известной высоты;
- экспорт итогового слоя с финальной высотой.

## Что делает проект
Пайплайн берет наборы зданий из разных источников, приводит их к общей структуре, объединяет записи, а затем строит модель, которая предсказывает высоту там, где она отсутствует.

Итог:
- для зданий с известной высотой сохраняется исходное значение;
- для зданий без высоты вычисляется `predicted_height`;
- в результирующем наборе формируется единое поле `final_height`.

## Архитектура пайплайна
```python
gdf_unified

    ↓

    ↓

    ↓
Экспорт итогового GeoPackage
```

## Tech Stack
Код написан в Jupyter notbook, в формате .ipynb.

Используется Python 3.10+

Необходимы библиотеки для установки:
`geopandas` — работа с геопространственными данными;

`shapely` — геометрические операции с фигурами;

`pyproj` — преобразования систем координат;

`folium` — визуализация геоданных на интерактивных картах;

`scikit-learn` — классические алгоритмы машинного обучения;

`lightgbm` — градиентный бустинг от Microsoft;

`catboost` — градиентный бустинг от Яндекса;

`matplotlib` — базовая библиотека для построения графиков;

`seaborn` — расширенная визуализация статистических данных;

`networkx` — анализ графов и сетей;

`rtree` — индекс для быстрого поиска пространственных данных.

В текущей версии используются два источника как исходные файлы для анализа:

1. cup_it_example_src_A.csv
2. cup_it_example_src_B.csv

Ожидается, что источники содержат:
- геометрию здания (geometry или wkt);
- признаки здания;
- высоту (height) — если она известна;
- другие атрибуты.

Основные этапы обработки
1. Загрузка данных

Источники читаются и приводятся к единому формату.

2. Очистка

Выполняется:
- парсинг WKT/geometry;
- отбрасывание пустых и некорректных геометрий;
- фильтрация объектов по площади;
приведение к единой CRS.

3. Пространственное сопоставление

Контуры из разных источников сопоставляются по пересечению и IoU, чтобы определить, какие записи относятся к одному и тому же зданию.

4. Выбор высоты

Если для одного здания есть несколько кандидатов по высоте, применяется приоритет источников и правила выбора наиболее надежного значения.

5. Разделение выборок

Формируются:
- train_eval_set — здания с известной высотой;
- predict_set — здания, где высоту нужно предсказать.

6. Feature Engineering

Используются два класса признаков:
- Безопасные до split
- площадь;
- периметр;
- компактность;
- центроид;
- размеры bbox;
- прямоугольность;
- вытянутость.

Только после fit на train fold
- grid statistics;
- KNN-признаки;
- priors по типу здания;
- кодирование категорий.

7. Обучение моделей

Сравниваются:
LightGBM;
CatBoost;
baseline-подходы.

Оценка проводится через spatial cross-validation.

8. Финальное обучение

После OOF-оценки лучшая модель обучается на всех известных высотах и применяется к predict_set.

9. Экспорт

Результат сохраняется в GeoPackage:
spb_buildings_final.gpkg

*Защита от data leakage

В проекте отдельно проработаны меры против leakage:
predict_set не участвует в обучении и оценке;
используется Spatial GroupKFold, а не случайный split;
все статистики, завязанные на target, считаются только на train_fold;
KNN для train строится в режиме leave-one-out;
признаки с прямой утечкой исключаются из обучения.
Примеры источников leakage, которые устранены
self-lookup в KNN;
grid stats на всем датасете до split;
type prior на всем датасете;
случайное разбиение соседних объектов в разные выборки;
использование stairs как почти прямого proxy для target.

- Метрики, используемые для оценки качества полученных данных:
MAE;
RMSE;
R²;
MAPE;
MedAE;
P90.

Это позволяет оценивать модель не только по средней ошибке, но и по устойчивости к выбросам и качеству на “тяжелых” объектах.

## Результат
На выходе формируется слой зданий с ключевыми полями:
`building_id; height; predicted_height; final_height; height_source; area_m2; centroid_lat; centroid_lon; bld_type; geometry`.

Логика итоговой высоты:
если `height` известна → используется она;
если `height` отсутствует → используется `predicted_height`;
результат записывается в `final_height`.
